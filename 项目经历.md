## 项目经历

1.other-info low-code 表单项目：

```
我们是一个做旅行的公司，主要推出各种活动让用户购买。用户购买之后在结算页就会填各种信息，不同的活动又需要填不同的表单，如果所以需要一个low code来自动生成表单而不是针对每个活动都开发一个表单，同时由于数据有了统一的结构，在后端就更容易对接。我主要负责low code这一块的工作，和后端讨论数据结构，把数据结构渲染成表单。由于我们业务的特殊性，我没有使用现有的库，而是自己写了一个表单渲染器，这个表单渲染器支持嵌套表单、表单校验、表单回填、用户便捷操作等功能。最后的结果是上架新活动的效率提高了，并且结算页生单的转化率提高了7个点。
```

2.库存重构项目：

```
库存重构是一个b端的项目，当时的背景是我们公司在活动页里面是卖套餐给用户的，套餐里面又有很多sku级别，比如说套餐是迪士尼玩一天，然后套餐下面就会有儿童票、成人票，这个儿童票和成人票就是sku了，这个和市面上其它旅游公司一样。但是我们的库存是套餐维度的，没有细到sku维度，但是实际商家售卖的是儿童票成人票这些sku，他们的库存是在sku维度，这就导致了如果多个套餐共用sku的时候，他们互相不知道对方的sku的售卖情况，所以需要商家手动调整套餐的库存从而间接调整sku的库存。所以我们的任务是前后端合作，把库存和价格细分到sku维度，从而简化商家的操作，并且能更方便的和商家对接。我负责对应的前端开发工作。我主要实现了一个和google日历差不多的日历，它支持常规的日期切换、新增timeslot和修改某个timeslot里面的库存和价格：成本价售卖价特殊价格等，并且支持不同的权限，最后成功上线了。由于这个只是第一期基础部分，第二期会做一些监控报警方面的优化，所以并没有相关的数据统计结果。
```
